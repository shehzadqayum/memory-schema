## Reasoning

Method: 3 web-research lenses (agentic-memory landscape, RAG/context-efficiency, LLM-compliance mechanics) + 2 code-grounded analysis lenses (schema spec, injection audit) + a first-person compliance self-audit (I am the injection mechanism's target — the audit of where the rules fail ME is primary evidence no external agent could produce). The decisive finding is a convergence FROM THREE INDEPENDENT DIRECTIONS on the same conclusion: the field's practice (no serious 2025+ system has the LLM hand-author escaped XML), the compliance literature (IFScale omission-decay + Anthropic's advisory-vs-deterministic doctrine), and our own measurements (code-enforced rules ~100% compliance, prose-enforced 50-100% violation) all say the same thing — the LLM should supply CONTENT, code should supply STRUCTURE. That triangulation is what elevates the chain-step CLI from an ergonomic nicety to the central architectural correction. Equally important is what the research VALIDATED: the md-file substrate, the L0 index, the chain-as-note-taking concept, and the retrieval quality (93% strong hits at a scale below where any advanced RAG technique differentiates) — so v5 is a format-and-governance slim-down, not a rebuild; the storage layers and recall loop survive intact. The spec self-contradictions (desc-120 vs evolving-summary; chain protocol vs Q6) explain the worst compliance numbers better than LLM negligence does: an unfollowable rule pair produces 91% violation regardless of model quality — fix the spec, not the model. Sequencing puts prose-deletion AFTER the deterministic replacement is proven, because the only genuinely dangerous step is a governance gap between the old prompt-protocol and the new code-protocol.

## Reasoning

Method: measurement-first, then adversarial code verification. Ran the full 764-test suite (green), all four eval modes (retrieval/salience/ablation/backends), and instrumented every live layer in-process (import cost, per-space embed, JSONL rewrite, bolt fetch profile, hook core, preflight) before any code-lens analysis — so the 30-finding workflow (9 verified before a session limit; the unverified remainder corroborated by my own measurements) argued from ground truth rather than speculation. The single most valuable discovery is that four separately-observed problems (session recall misses, Re-embedded:0, the 19K description, multi-space negative lift) all reduce to ONE line — head-truncating [:2000] composition — plus the absence of any embedding-provenance hash: derived state silently decouples from source state as entities grow. The second insight is that the system's failures this month were all CAUGHT by its file-first core (reconcile guard, md recovery) and MISSED by its derived layers — which is exactly the right failure direction, and why the plan hardens the trust loop (Phase 0) before touching quality or speed. Pre-committed decision rules were honoured where they existed (ablation dormant verdict → a real off-switch) and added where missing (Neo4j re-test after the hot-path fix, not before — fixing the 78%-overhead fetch may flip the backends verdict).
